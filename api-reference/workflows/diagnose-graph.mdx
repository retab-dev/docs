---
title: "Diagnose Workflow Graph"
openapi: "POST /v1/workflows/{workflow_id}/diagnose-graph"
---

Validate workflow blocks and edges before publishing. You can pass a draft
graph directly to this endpoint.

The check covers:

- Missing or duplicate start_document blocks
- Disconnected blocks
- Dangling edges (one endpoint missing)
- Unreachable blocks
- Type mismatches between source/target handles
- Missing required block configuration
- Incomplete authored config such as blank categories, subdocuments, or map keys
- Embedded review gate predicates and unavailable review signals
- Cycles in the directed graph

Severities:

| Severity  | Meaning                                          |
| --------- | ------------------------------------------------ |
| `error`   | Must fix before publish — block runs would fail. |
| `warning` | Should fix; runs may still succeed.              |
| `info`    | Advisory only.                                   |

`is_valid` is `true` when the issue list contains no errors. `re_propagate`
defaults to `true` so derived schemas are refreshed before validation.

The SDKs also expose a convenience helper:

<RequestExample>

```python Python
from retab import Retab

client = Retab()

diagnosis = client.workflows.diagnose("wf_abc123")
for issue in diagnosis.issues:
    print(f"[{issue.severity}] {issue.code}: {issue.message}")
```

```go Go
package main

import (
	"context"
	"fmt"
	"log"

	retab "github.com/retab-dev/retab/clients/go"
)

func main() {
	ctx := context.Background()

	client, err := retab.NewClient("")
	if err != nil {
		log.Fatal(err)
	}

	diagnosis, err := client.Workflows.Diagnose(ctx, "wf_abc123")
	if err != nil {
		log.Fatal(err)
	}

	for _, issue := range diagnosis.Issues {
		fmt.Printf("[%s] %s: %s\n", issue.Severity, issue.Code, issue.Message)
	}
}
```

```rust Rust
use retab::models::WorkflowGraphDiagnosisRequest;
use retab::resources::workflows::DiagnoseParams;
use retab::Retab;

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {
    let client = Retab::new(std::env::var("RETAB_API_KEY")?);

    let diagnosis = client
        .workflows()
        .diagnose(
            "wf_abc123",
            DiagnoseParams::new(WorkflowGraphDiagnosisRequest::default()),
        )
        .await?;

    for issue in diagnosis.issues.unwrap_or_default() {
        println!("[{}] {}: {}", issue.severity, issue.code, issue.message);
    }
    Ok(())
}
```

```curl cURL
# Direct call: pass the graph payload yourself.
curl -X 'POST' \
  'https://api.retab.com/v1/workflows/wf_abc123/diagnose-graph' \
  -H 'Content-Type: application/json' \
  -H 'Api-Key: <your-api-key>' \
  -d '{
    "blocks": [
      {
        "id": "start-1",
        "type": "start_document",
        "label": "Start",
        "config": null,
        "position": { "x": 0, "y": 0 }
      }
    ],
    "edges": [],
    "re_propagate": true
  }'
```
