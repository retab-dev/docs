---
title: 'Get Workflow Entities'
openapi: 'GET /v1/workflows/{workflow_id}/entities'
---

Fetch a workflow together with its full graph: every block and edge in the current draft.

This is the easiest way to discover the IDs you need for other endpoints — most importantly, the `start` and `start_json` block IDs you must pass to [Run Workflow](/api-reference/workflows/runs/create).

The Python SDK exposes two convenience properties on the response:

- `start_blocks` — document-input blocks (`type == "start"`)
- `start_json_blocks` — JSON-input blocks (`type == "start_json"`)

<RequestExample>

```python python
from retab import Retab

client = Retab()

graph = client.workflows.get_entities("wf_abc123xyz")

print(graph.workflow.name)
print(f"{len(graph.blocks)} blocks, {len(graph.edges)} edges")

for block in graph.start_blocks:
    print(f"document input: {block.id}")

for block in graph.start_json_blocks:
    print(f"json input: {block.id}")
```

```javascript javascript
import { Retab } from '@retab/node';

const client = new Retab();

const graph = await client.workflows.getEntities("wf_abc123xyz");

console.log(graph.workflow.name);
console.log(`${graph.blocks.length} blocks, ${graph.edges.length} edges`);

const startBlocks = graph.blocks.filter((b) => b.type === "start");
for (const block of startBlocks) {
  console.log(`document input: ${block.id}`);
}
```

```go go
package main

import (
	"context"
	"fmt"
	"log"

	retab "github.com/retab-dev/retab/clients/go"
)

func main() {
	ctx := context.Background()

	client, err := retab.NewClient("")
	if err != nil {
		log.Fatal(err)
	}

	graph, err := client.Workflows.GetEntities(ctx, "wf_abc123xyz")
	if err != nil {
		log.Fatal(err)
	}

	fmt.Println(graph.Workflow.Name)
	fmt.Printf("%d blocks, %d edges\n", len(graph.Blocks), len(graph.Edges))

	for _, block := range graph.Blocks {
		if block.Type == "start" {
			fmt.Printf("document input: %s\n", block.ID)
		}
		if block.Type == "start_json" {
			fmt.Printf("json input: %s\n", block.ID)
		}
	}
}
```
