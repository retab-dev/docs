---
title: "Get Test Run Result"
openapi: "GET /v1/workflows/tests/results/{result_id}"
---

Fetch one workflow-test result by its opaque `result_id`. Use this to refresh a
single result row without re-listing every result for the parent test run.

<RequestExample>

```python Python
from retab import Retab

client = Retab()

result = client.workflows.tests.results.get("wfnodetestrun_a")

print(result.test_id)
print(result.lifecycle.status)
print(result.verdict)
```

```typescript TypeScript
import { Retab } from "@retab/node";

const client = new Retab({ apiKey: process.env.RETAB_API_KEY });

const result = await client.workflows.tests.results.get("wfnodetestrun_a");
console.log(result.testId);
console.log(result.lifecycle.status);
console.log(result.verdict);
```

```go Go
package main

import (
	"context"
	"fmt"
	"log"

	retab "github.com/retab-dev/retab/clients/go"
)

func main() {
	ctx := context.Background()

	client, err := retab.NewClient("")
	if err != nil {
		log.Fatal(err)
	}

	result, err := client.Workflows.Tests.Results.Get(ctx, "wfnodetestrun_a")
	if err != nil {
		log.Fatal(err)
	}

	fmt.Println((*result)["test_id"])
	fmt.Println((*result)["lifecycle"])
	fmt.Println((*result)["verdict"])
}
```

```rust Rust
use retab::Retab;

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {
    let client = Retab::new(std::env::var("RETAB_API_KEY")?);

    let result = client
        .workflow_test_run_results()
        .get("wfnodetestrun_a")
        .await?;
    println!("{}", result.test_id);
    println!("{:?}", result.lifecycle);
    println!("{:?}", result.verdict);
    Ok(())
}
```
