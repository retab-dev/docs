---
title: 'Execute Workflow Tests'
openapi: 'POST /v1/workflows/{workflow_id}/tests/runs'
---

Run one workflow test, all tests for a single block, or every test in a workflow.
Execution is asynchronous: the response returns immediately with a
`run_id`. Poll the workflow-test run until it reaches `completed` or `failed`,
then read the run results from the workflow-test runs endpoint.

The request body provides EXACTLY ONE of:

- **`test_id`** — run a single test by id.
- **`target`** — run all tests for a single block (`{ type: "block", block_id: ... }`).
- *neither* — run every test in the workflow.

`n_consensus` is optional and only meaningful for extract / split / classifier
blocks. Allowed values are `3`, `5`, or `7`. Provide it to override the
block's configured consensus count for this run only.

<RequestExample>

```python python
from retab import Retab

client = Retab()

# Single test
run = client.workflows.tests.execute(
    workflow_id="wf_abc123xyz",
    test_id="wfnodetest_hsLEQiM61ez9Piv147MWk",
)

# All tests for a single block
block_run = client.workflows.tests.execute(
    workflow_id="wf_abc123xyz",
    target={"type": "block", "block_id": "block_extract_invoice"},
    n_consensus=5,
)

# Every test in the workflow
all_run = client.workflows.tests.execute(
    workflow_id="wf_abc123xyz",
)

print(f"Workflow-test run {run.run_id} queued")
```

## Polling for the run

Poll the workflow-test run by `run_id` until terminal. When the run completes,
fetch its results from `/tests/runs/{run_id}/results`.

<CodeGroup>
```python Python
import time

run = client.workflows.tests.execute(workflow_id="wf_abc123xyz")
deadline = time.monotonic() + 600
while True:
    status = client.workflows.tests.runs.get_execution(
        workflow_id="wf_abc123xyz",
        run_id=run.run_id,
    )
    if status.status in ("completed", "failed"):
        break
    if time.monotonic() >= deadline:
        raise TimeoutError(f"Timed out waiting for workflow test run {run.run_id}")
    time.sleep(2)

if status.status != "completed":
    raise RuntimeError(f"Workflow test run failed: {status.error}")

result = client.workflows.tests.runs.results(
    workflow_id="wf_abc123xyz",
    run_id=run.run_id,
)
print(f"{result.counts.passed}/{result.counts.passed + result.counts.failed} passed")
for item in result.results:
    print(item.test_id, item.status, item.duration_ms)
```

```javascript JavaScript
const run = await client.workflows.tests.execute({
  workflowId: "wf_abc123xyz",
});

const deadlineMs = Date.now() + 600_000;
let status;
while (true) {
  status = await client.workflows.tests.runs.getExecution({
    workflowId: "wf_abc123xyz",
    runId: run.run_id,
  });
  if (["completed", "failed"].includes(status.status)) {
    break;
  }
  if (Date.now() >= deadlineMs) {
    throw new Error(`Timed out waiting for workflow test run ${run.run_id}`);
  }
  await new Promise((resolve) => setTimeout(resolve, 2000));
}

if (status.status !== "completed") {
  throw new Error(`Workflow test run failed: ${status.error}`);
}

const result = await client.workflows.tests.runs.results({
  workflowId: "wf_abc123xyz",
  runId: run.run_id,
});
console.log(`${result.counts.passed}/${result.results.length} passed`);
for (const item of result.results) {
  console.log(item.test_id, item.status, item.duration_ms);
}
```

```go Go
package main

import (
	"context"
	"fmt"
	"log"
	"time"

	retab "github.com/retab-dev/retab/clients/go"
)

func main() {
	ctx := context.Background()

	client, err := retab.NewClient("")
	if err != nil {
		log.Fatal(err)
	}

	run, err := client.Workflows.Tests.Execute(ctx, retab.ExecuteWorkflowTestsRequest{
		WorkflowID: "wf_abc123xyz",
	})
	if err != nil {
		log.Fatal(err)
	}

	deadline := time.Now().Add(10 * time.Minute)
	var status *retab.WorkflowTestExecutionRunStatus
	for {
		status, err = client.Workflows.Tests.Runs.GetExecution(ctx, "wf_abc123xyz", run.RunID, nil)
		if err != nil {
			log.Fatal(err)
		}
		if status.Status == "completed" || status.Status == "failed" {
			break
		}
		if time.Now().After(deadline) {
			log.Fatalf("timed out waiting for workflow test run %s", run.RunID)
		}
		time.Sleep(2 * time.Second)
	}
	if status.Status != "completed" {
		log.Fatalf("workflow test run failed: %s", status.Error)
	}

	result, err := client.Workflows.Tests.Runs.Results(ctx, "wf_abc123xyz", run.RunID, nil)
	if err != nil {
		log.Fatal(err)
	}
	fmt.Printf("%d/%d passed\n", result.Counts.Passed, len(result.Results))
	for _, item := range result.Results {
		fmt.Println(item.ID, item.Status, item.DurationMs)
	}
}
```
</CodeGroup>

Treat `failed` as a terminal failure. If the polling
deadline elapses before a terminal status, raise or return a timeout from your
application code.

## What the run results contain

After the workflow-test run reaches `completed`, the results endpoint returns:

```json
{
  "workflow_id": "wf_abc123xyz",
  "target": { "type": "block", "block_id": "block_extract_invoice" },
  "counts": {
    "queued": 0,
    "running": 0,
    "passed": 3,
    "failed": 1,
    "blocked": 0,
    "error": 0,
    "cancelled": 0
  },
  "results": [
    {
      "test_id": "wfnodetest_a",
      "run_record_id": "wfnodetestrun_a",
      "status": "passed",
      "workflow_id": "wf_abc123xyz",
      "target": { "type": "block", "block_id": "block_extract_invoice" },
      "duration_ms": 18221
    }
  ]
}
```

Each `WorkflowTestBatchExecutionItem` carries the `run_record_id` you can pass to
[Get Workflow Test Run](./runs/get) to fetch the full execution snapshot
(inputs, `outputs`, `assertion_result`, `verdict_summary`, etc.).

The `counts` object has one bucket per [run-record status value](/workflows/tests-api#run-record-status).
