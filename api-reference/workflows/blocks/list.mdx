---
title: 'List Blocks'
openapi: 'GET /v1/workflows/{workflow_id}/blocks'
---

List every block in a workflow's current draft.

The response carries `resolved_schemas` on each block — the graph-derived input/output JSON schemas the engine will use at runtime. This is the source of truth for handle shapes; raw block configs may not include them directly.

For the full graph (blocks + edges in a single call), use [Get Workflow Entities](/api-reference/workflows/get-entities) instead.

<RequestExample>

```python python
from retab import Retab

client = Retab()

blocks = client.workflows.blocks.list("wf_abc123xyz")

for block in blocks:
    print(f"{block.id} ({block.type}): {block.label}")
```

```javascript javascript
import { Retab } from '/node';

const client = new Retab();

const blocks = await client.workflows.blocks.list("wf_abc123xyz");

for (const block of blocks) {
  console.log(`${block.id} (${block.type}): ${block.label}`);

```

```go go
package main

import (
	"context"
	"fmt"
	"log"

	retab "github.com/retab-dev/retab/clients/go"
)

func main() {
	ctx := context.Background()

	client, err := retab.NewClient("")
	if err != nil {
		log.Fatal(err)
	}

	blocks, err := client.Workflows.Blocks.List(ctx, "wf_abc123xyz")
	if err != nil {
		log.Fatal(err)
	}

	for _, block := range blocks {
		fmt.Printf("%s (%s): %s\n", block.ID, block.Type, block.Label)
	}
}
```
