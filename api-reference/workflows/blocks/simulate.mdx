---
title: 'Simulate Block'
openapi: 'POST /v1/workflows/runs/{run_id}/steps/{block_id}/simulate'
---

Replay one workflow block using inputs from a previous run plus the **current
draft block config**. The block executes once but its result is not persisted
into the run — useful for testing config changes without re-running the whole
workflow.

The route is keyed by `run_id` (the workflow run whose recorded inputs feed
the block), not by `workflow_id`.

Behavior:

1. Looks up the run and verifies the upstream subgraph hasn't drifted since
   it executed (skip with `check_eligibility=false`).
2. Fetches the block's `handle_inputs` from the run's recorded step data.
3. Loads the block's **current draft config** (not the published snapshot).
4. Executes the block.
5. Returns the produced outputs and a canonical `artifact` ref where the
   block produces a persisted resource (extract → extraction, split → split,
   etc.).

Query params:

- **`n_consensus`** — `3` / `5` / `7`. Only meaningful for `extract`,
  `split`, and `classifier` blocks; ignored otherwise.
- **`step_id`** — for blocks inside a `for_each` or `while_loop`, pick a
  specific iteration. Defaults to the base step or the first iteration.
- **`check_eligibility`** — defaults to `true`. Returns `409` if the
  upstream subgraph changed since `run_id` executed. Pass `false` to skip
  the check.

Unsupported block types: `start`, `start_json`, `api_call`, plus
display-only blocks like `note`.

<RequestExample>

```python python
from retab import Retab

client = Retab()

simulation = client.workflows.blocks.simulate(
    run_id="wfrun_def456",
    block_id="extract-invoice",
    n_consensus=5,
)

if simulation.success:
    print(simulation.handle_outputs)
else:
    print(simulation.error)
```

```javascript javascript
import { Retab } from '@retab/node';

const client = new Retab();

const simulation = await client.workflows.blocks.simulate({
```

```go go
package main

import (
	"context"
	"fmt"
	"log"

	retab "github.com/retab-dev/retab/clients/go"
)

func main() {
	ctx := context.Background()

	client, err := retab.NewClient("")
	if err != nil {
		log.Fatal(err)
	}

	simulation, err := client.Workflows.Blocks.Simulate(ctx, retab.SimulateBlockRequest{
		RunID:      "wfrun_def456",
		BlockID:    "extract-invoice",
		NConsensus: 5,
	})
	if err != nil {
		log.Fatal(err)
	}

	if simulation.Success {
		fmt.Println(simulation.HandleOutputs)
	} else {
		fmt.Println(simulation.Error)
	}
}
```

```bash curl
# Default — uses the run's inputs and the current draft block config.
curl -X 'POST' \
  'https://api.retab.com/v1/workflows/runs/wfrun_def456/steps/extract-invoice/simulate' \
  -H 'Api-Key: <your-api-key>'

# With consensus override + iteration step pick.
curl -X 'POST' \
  'https://api.retab.com/v1/workflows/runs/wfrun_def456/steps/extract-invoice/simulate?n_consensus=7&step_id=for_each-0/extract-invoice-0' \
  -H 'Api-Key: <your-api-key>'

# Skip the upstream-drift check.
curl -X 'POST' \
  'https://api.retab.com/v1/workflows/runs/wfrun_def456/steps/extract-invoice/simulate?check_eligibility=false' \
  -H 'Api-Key: <your-api-key>'
```
