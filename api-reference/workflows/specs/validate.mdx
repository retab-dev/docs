---
title: 'Validate Workflow Spec'
openapi: 'POST /v1/workflows/yaml/validate'
---

Validate a declarative workflow YAML spec without changing workflow state.

Declarative specs use `apiVersion: workflows.retab.com/v1alpha2`. Edges must be explicit and handle-based:

```yaml
spec:
  edges:
    - from:
        block: start-node
        handle: output-file-0
      to:
        block: extract-node
        handle: input-file-source_doc
```

Handles are raw runtime handles and are validated against the block type and block configuration.

<RequestExample>

```python python
from retab import Retab

client = Retab()

validation = client.workflows.specs.validate(yaml_definition)

print(validation.is_valid)
print(validation.diagnostics)
```

```javascript javascript
import { Retab } from '/node';

const client = new Retab();

const validation = await client.workflows.specs.validate(yamlDefinition);

console.log(validation.is_valid);
console.log(validation.diagnostics);
```

```bash curl
curl -X 'POST' \
  'https://api.retab.com/v1/workflows/yaml/validate' \
  -H 'Content-Type: application/json' \
  -H 'Api-Key: <your-api-key>' \
  -d '{
    "yaml_definition": "apiVersion: workflows.retab.com/v1alpha2\nkind: Workflow\nmetadata:\n  id: wf_abc123\n  name: Invoice Workflow\nspec:\n  blocks:\n    start-node:\n      type: start\n      label: Input Document\n    extract-node:\n      type: extract\n      label: Extract Fields\n      config:\n        inputs:\n          - name: source_doc\n            type: file\n            is_primary: true\n        json_schema:\n          type: object\n          properties: {}\n  edges:\n    - from:\n        block: start-node\n        handle: output-file-0\n      to:\n        block: extract-node\n        handle: input-file-source_doc\n"
  }'
```
