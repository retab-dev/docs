---
title: 'Apply Workflow Spec'
openapi: 'POST /v1/workflows/spec/apply'
---

Apply a declarative workflow YAML spec to draft workflow state.

`apply()` writes the canonical draft graph represented by the YAML spec. It does not publish. Call the workflow publish method separately when the draft is ready to become the live version.

<RequestExample>

```python python
from retab import Retab

client = Retab()

result = client.workflows.specs.apply(yaml_definition)

print(result.workflow_id)
print(result.created)
print(result.summary.has_changes)
```

```javascript javascript
import { Retab } from '@retab/node';

const client = new Retab();

const result = await client.workflows.specs.apply(yamlDefinition);

console.log(result.workflow_id);
console.log(result.created);
console.log(result.summary.has_changes);
```

```bash curl
curl -X 'POST' \
  'https://api.retab.com/v1/workflows/spec/apply' \
  -H 'Content-Type: application/json' \
  -H 'Api-Key: <your-api-key>' \
  -d '{
    "yaml_definition": "<workflow-yaml>"
  }'
```
