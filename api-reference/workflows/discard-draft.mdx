---
title: "Discard Draft"
openapi: "POST /v1/workflows/{workflow_id}/discard-draft"
---

Discard all unpublished draft changes and restore the workflow to its last published version. The blocks and edges are recreated from the published snapshot, so the draft you see in the editor matches what new runs execute.

This is the inverse of [Publish Workflow](/api-reference/workflows/publish): publish promotes the draft to a new version; discard throws the draft away and rolls back to the current published version. The workflow must already be published — there is no published state to restore to otherwise.

<RequestExample>

```python Python
from retab import Retab

client = Retab()

workflow = client.workflows.discard_draft("wf_abc123xyz")

print(workflow.published.version_id)
```

```go Go
package main

import (
	"context"
	"fmt"
	"log"

	retab "github.com/retab-dev/retab/clients/go"
)

func main() {
	ctx := context.Background()

	client, err := retab.NewClient("")
	if err != nil {
		log.Fatal(err)
	}

	workflow, err := client.Workflows.DiscardDraft(ctx, "wf_abc123xyz")
	if err != nil {
		log.Fatal(err)
	}

	if workflow.Published != nil {
		fmt.Println(workflow.Published.VersionID)
	}
}
```

```rust Rust
    }
```
