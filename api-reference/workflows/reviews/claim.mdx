---
title: 'Claim Review'
openapi: 'POST /v1/workflows/reviews/{run_id}/{block_id}/claim'
---

Take the advisory review claim on an overlay.

The claim records, in `claim`, a `ReviewClaim` (`{holder, claimed_at, expires_at}`) naming the `Actor` currently working a review. It is **advisory** — it does not lock the overlay or prevent another actor from editing or deciding. It exists so a review inbox can show "Dana is reviewing this" and avoid two people picking up the same item. The hard concurrency guarantee comes from the `version_stamp` compare-and-swap on every mutating call, not from the claim.

The request body carries only the `version_stamp` you last read.

<Warning>
`version_stamp` is a compare-and-swap token. Send the value you last read from [Get Review](/api-reference/workflows/reviews/get). If the overlay has advanced, the call returns **`409`** — re-read the overlay and retry. A stale stamp raises `retab.exceptions.ConflictError` in Python and surfaces as an `APIError` with `.status === 409` in JavaScript.
</Warning>

Release the claim with [Release Review](/api-reference/workflows/reviews/release) when you step away without deciding. Submitting a decision does not require holding the claim.

<RequestExample>

```python python
from retab import Retab

client = Retab()

overlay = client.workflows.runs.reviews.get("run_abc123", "hil-1")

overlay = client.workflows.runs.reviews.claim(
    "run_abc123", "hil-1", version_stamp=overlay.version_stamp,
)
print("claimed by:", overlay.claim.holder.display_name)
```

```javascript javascript
import { Retab } from '@retab/node';

const client = new Retab();

let overlay = await client.workflows.runs.reviews.get("run_abc123", "hil-1");

overlay = await client.workflows.runs.reviews.claim("run_abc123", "hil-1", {
  versionStamp: overlay.rev,
});
console.log("claimed by:", overlay.claim?.holder?.display_name);
```

```bash curl
curl -X 'POST' \
  'https://api.retab.com/v1/workflows/reviews/run_abc123/hil-1/claim' \
  -H 'Content-Type: application/json' \
  -H 'Api-Key: <your-api-key>' \
  -d '{ "version_stamp": 3 }'
```

</RequestExample>
