---
title: 'List Reviews'
openapi: 'GET /v1/workflows/reviews'
---

List the review queue — every gated block run that has an open review.

Each item is a `ReviewQueueItem` summary, not the full review. Use [Get Review](/api-reference/workflows/reviews/get) to fetch `versions_by_id` and `decision` for a given `(run_id, block_id)` pair.

Filter the queue with the query parameters:

| Parameter | Notes |
|---|---|
| `workflow_id` | Optional — restrict to a single workflow. |
| `decision` | Defaults to `none`, which lists only reviews still awaiting a decision. Use `any` to include reviews that already have a terminal decision. |
| `before` | Optional cursor from `list_metadata.before` for the previous page. |
| `after` | Optional cursor from `list_metadata.after` for the next page. |
| `limit` | `1`–`200`. Defaults to a server-side page size. `list_metadata.after != null` means another page is available. |

<Note>
This is the entry point for building a review inbox. For the end-to-end loop — editing and submitting a verdict — see the [Reviews guide](/workflows/Reviews).
</Note>

<RequestExample>

```python python
from retab import Retab

client = Retab()

queue = client.workflows.reviews.list(workflow_id="wf_1", limit=50)

for item in queue.data:
    print(item.workflow_run_id, item.block_id, item.awaiting_since)

if queue.list_metadata.after:
    next_page = client.workflows.reviews.list(after=queue.list_metadata.after)
```

```javascript javascript
import { Retab } from '@retab/node';

const client = new Retab();

const queue = await client.workflows.reviews.list({
  workflowId: "wf_1",
  limit: 50,
});

for (const item of queue.data) {
  console.log(item.workflow_run_id, item.block_id, item.awaiting_since);
}

if (queue.list_metadata.after) {
  const nextPage = await client.workflows.reviews.list({
    after: queue.list_metadata.after,
  });
}
```

```bash curl
curl -X 'GET' \
  'https://api.retab.com/v1/workflows/reviews?workflow_id=wf_1&limit=50' \
  -H 'Api-Key: <your-api-key>'
```
