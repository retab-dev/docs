---
title: 'Run Experiments for Block'
openapi: 'POST /v1/workflows/{workflow_id}/experiments/run-batch'
---

Trigger a run for every experiment attached to one block, in one call.

Useful when you've made a change to a block config and want to refresh all
experiments attached to it without iterating client-side. Each experiment
returns the same payload as
[Run Experiment](/api-reference/workflows/experiments/runs/create).

Returns `404` if no experiments are attached to the block.

<RequestExample>

```python python
from retab import Retab

client = Retab()

result = client.workflows.experiments.run_batch(
    workflow_id="wf_abc123",
    block_id="extract-invoice",
    n_consensus=5,
)
print(f"Triggered {result.experiment_count} experiments")
for run in result.runs:
    print(run.experiment_id, run.run_id, run.status)
```


```go go
package main

import (
	"context"
	"fmt"
	"log"

	retab "github.com/retab-dev/retab/clients/go"
)

func main() {
	ctx := context.Background()

	client, err := retab.NewClient("")
	if err != nil {
		log.Fatal(err)
	}

	result, err := client.Workflows.Experiments.RunBatch(ctx, retab.RunBatchExperimentsRequest{
		WorkflowID: "wf_abc123",
		BlockID:    "extract-invoice",
		NConsensus: 5,
	})
	if err != nil {
		log.Fatal(err)
	}

	fmt.Printf("Triggered %d experiments\n", result.ExperimentCount)
	for _, run := range result.Runs {
		fmt.Println(run.ExperimentID, run.RunID, run.JobID)
	}
}
```

