---
title: 'List Eligible Blocks'
openapi: 'GET /v1/workflows/{workflow_id}/experiments/eligible-blocks'
---

List blocks in the current draft that can host experiments — `extract`,
`split`, `classifier`, and `for_each` blocks configured with
`map_method="split_by_key"`. Each row also carries roll-ups for the
experiments already attached to that block (counts, mean score, latest run).

Useful for building a dashboard "add experiment" picker without separately
listing blocks and then filtering client-side.

<RequestExample>

```python python
from retab import Retab

client = Retab()

result = client.workflows.experiments.list_eligible_blocks(
    workflow_id="wf_abc123",
)
for block in result.blocks:
    print(block.block_id, block.block_type, block.experiment_count)
```


```go go
package main

import (
	"context"
	"fmt"
	"log"

	retab "github.com/retab-dev/retab/clients/go"
)

func main() {
	ctx := context.Background()

	client, err := retab.NewClient("")
	if err != nil {
		log.Fatal(err)
	}

	result, err := client.Workflows.Experiments.ListEligibleBlocks(ctx, "wf_abc123")
	if err != nil {
		log.Fatal(err)
	}

	for _, block := range result.Blocks {
		fmt.Println(block.BlockID, block.BlockType, block.ExperimentCount)
	}
}
```
