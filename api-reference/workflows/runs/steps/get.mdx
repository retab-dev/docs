---
title: 'Get Step'
openapi: 'GET /v1/workflows/runs/{run_id}/steps/{block_id}'
---

Get step status and handle data for a specific step in a workflow run. Use this to inspect the normalized inputs and outputs for a particular block after or during execution.

<RequestExample>

```python python
from retab import Retab

client = Retab()

step = client.workflows.runs.steps.get("run_abc123xyz", "extract-block-1")

print(step.status)
payload = step.execution.outputs.get("output-json-0")
print(payload.data if payload and payload.type == "json" else None)

# Jump to the underlying typed resource produced by this step:
extraction_ref = next(
    (artifact for artifact in step.execution.artifacts if artifact.operation == "extraction"),
    None,
)
if extraction_ref:
    extraction = client.extractions.get(extraction_ref.id)
    print(extraction.choices)

# Or inspect raw handle payloads:
if step.execution.outputs:
    payload = step.execution.outputs["output-json-0"]
    print(payload.type)
    print(payload.data)
```

```javascript javascript
import Retab from 'retab';

const client = new Retab();

const step = await client.workflows.runs.steps.get("run_abc123xyz", "extract-block-1");

console.log(`Step status: ${step.status}`);
const extractionRef = step.execution.artifacts.find((artifact) => artifact.operation === "extraction");
if (extractionRef) {
    const extraction = await client.extractions.get(extractionRef.id);
    console.log(extraction);
}
```

```bash curl
curl -X 'GET' \
  'https://api.retab.com/v1/workflows/runs/run_abc123xyz/steps/extract-block-1' \
  -H 'Api-Key: <your-api-key>'
```
