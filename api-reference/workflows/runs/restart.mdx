---
title: 'Restart Run'
openapi: 'POST /v1/workflows/runs/{run_id}/restart'
---

Restart a completed, errored, or cancelled run with the same inputs. A brand-new run is created with `lifecycle.kind` set to `running`; the original run is left untouched.

The restart picks up the current published snapshot — not the snapshot the original run executed against. If you've published changes since the original run, the restart reflects them.

`command_id` is an optional idempotency key — replaying the same value never enqueues a second restart.

<RequestExample>

```python python
from retab import Retab

client = Retab()

new_run = client.workflows.runs.restart(
    run_id="run_abc123",
    command_id="cmd_restart_abc",  # optional dedup key
)

print(new_run.id)        # new run ID
print(new_run.lifecycle.kind)    # "running"
```

```javascript javascript
import { Retab } from '/node';

const client = new Retab();

const newRun = await client.workflows.runs.restart("run_abc123", {
  commandId: "cmd_restart_abc",
});

console.log(newRun.id);

```

```go go
package main

import (
	"context"
	"fmt"
	"log"

	retab "github.com/retab-dev/retab/clients/go"
)

func main() {
	ctx := context.Background()

	client, err := retab.NewClient("")
	if err != nil {
		log.Fatal(err)
	}

	newRun, err := client.Workflows.Runs.Restart(ctx, "run_abc123", retab.WorkflowRunCommandRequest{
		CommandID: "cmd_restart_abc", // optional dedup key
	})
	if err != nil {
		log.Fatal(err)
	}

	fmt.Println(newRun.ID)
	fmt.Println(newRun.Lifecycle.Kind)
}
```

```bash curl
curl -X 'POST' \

  -H 'Content-Type: application/json' \
  -H 'Api-Key: <your-api-key>' \
  -d '{"command_id": "cmd_restart_abc"}'
```
