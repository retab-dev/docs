---
title: 'Create Partitions'
openapi: 'POST /v1/partitions'
---

Partition a document into repeated chunks keyed by a value such as `invoice_number`, `policy_id`, or `claim_number`.

<RequestExample>

```python python
from retab import Retab

client = Retab()

response = client.partitions.create(
    document="invoice_batch.pdf",
    key="invoice_number",
    instructions="Return one chunk per invoice number and keep all pages for the same invoice together.",
    model="retab-small",
    n_consensus=3,
    bust_cache=False,
)

for chunk in response.output:
    print(chunk.key, chunk.pages)
```

```javascript javascript

for (const chunk of response.output) {

}
```

## Request Body

<ParamField body="document" type="MIMEData" required>
  The document to partition.
</ParamField>

<ParamField body="key" type="string" required>
  The field or concept used to group the document into chunks, such as `invoice_number` or `claim_id`.
</ParamField>

<ParamField body="instructions" type="string" required>
  Natural-language guidance describing how the document should be partitioned.
</ParamField>

<ParamField body="model" type="string" default="retab-small">
  The model used for partitioning.
</ParamField>

<ParamField body="n_consensus" type="integer" default="1">
  Number of partitioning runs to use for consensus voting. Max: `8`.
</ParamField>

<ParamField body="bust_cache" type="boolean" default="false">
  When `true`, bypass the cache and force a fresh partition run.
</ParamField>

## Response Fields

<ResponseField name="output" type="array[PartitionChunk]">
  The partitioned document chunks.
  <Expandable title="properties">
    <ResponseField name="key" type="string">
      The detected partition key value for this chunk.
    </ResponseField>
    <ResponseField name="pages" type="array[integer]">
      Pages (1-indexed) assigned to this chunk.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="consensus" type="PartitionConsensus">
  Consensus metadata for multi-vote runs.
  <Expandable title="properties">
    <ResponseField name="likelihoods" type="array[PartitionChunkLikelihood] | null">
      Likelihoods aligned positionally with `output`.
    </ResponseField>
    <ResponseField name="choices" type="array[array[PartitionChunk]]">
      Alternative partition vote outputs used to build the consolidated result.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="usage" type="RetabUsage | null">
  Usage information for the partition operation.
</ResponseField>
