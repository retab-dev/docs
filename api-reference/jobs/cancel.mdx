## Cancellation Behavior

The cancellation behavior depends on the job's current status:

| Current Status | Behavior                                                              |
| -------------- | --------------------------------------------------------------------- |
| `validating`   | Immediately cancelled                                                 |
| `queued`       | Removed from queue and cancelled                                      |
| `in_progress`  | Marked for cancellation; processing may complete if already finishing |
| `completed`    | Returns 400 error - cannot cancel                                     |
| `failed`       | Returns 400 error - cannot cancel                                     |
| `cancelled`    | Returns 400 error - already cancelled                                 |
| `expired`      | Returns 400 error - cannot cancel                                     |

<Note>
  Jobs that are `in_progress` may still complete if the cancellation request
  arrives after processing has finished but before the status is updated. Always
  check the final status of the returned job object.
</Note>

<Note>
  If cancellation races with a concurrent state transition, Retab may return
  `409`. In that case, retrieve the job again and decide whether another cancel
  attempt is still needed.
</Note>

## Use Cases

### Cancel a Batch of Jobs

<CodeGroup>
```python Python
# Cancel all queued jobs from a specific batch
jobs = client.jobs.list(status="queued")
for job in jobs.data:
    if job.metadata and job.metadata.get("batch_id") == "batch_001":
        try:
            client.jobs.cancel(job.id)
            print(f"Cancelled {job.id}")
        except Exception as e:
            print(f"Failed to cancel {job.id}: {e}")
```

```typescript TypeScript
// Cancel all queued jobs from a specific batch
const jobs = await client.jobs.list({ status: "queued" });

    try {
      await client.jobs.cancel(job.id);
      console.log(`Cancelled ${job.id}`);
    } catch (e) {
      console.log(`Failed to cancel ${job.id}:`, e);
    }
  }
```

```go Go
ctx := context.Background()
client, _ := retab.NewClient("")

// Cancel all queued jobs from a specific batch

	log.Fatal(err)

		fmt.Printf("Failed to cancel %s: %v\n", jobID, err)
		continue
	}
	fmt.Printf("Cancelled %s\n", jobID)
}
```

### Timeout-based Cancellation

<CodeGroup>
```python Python
import time

job = client.jobs.create(endpoint="/v1/extractions", request={...})
start_time = time.time()
timeout = 60 # 60 seconds

while job.status not in ("completed", "failed", "cancelled", "expired"):
if time.time() - start_time > timeout:
job = client.jobs.cancel(job.id)
print("Job cancelled due to timeout")
break
time.sleep(2)
job = client.jobs.get(job.id)

````

```typescript TypeScript
const job = await client.jobs.create("/v1/extractions", {...});
const startTime = Date.now();
const timeout = 60000;  // 60 seconds

const terminalStatuses = ["completed", "failed", "cancelled", "expired"];
while (!terminalStatuses.includes(job.status)) {
    if (Date.now() - startTime > timeout) {
        const cancelledJob = await client.jobs.cancel(job.id);
        console.log("Job cancelled due to timeout");
        break;
    }
    await new Promise(r => setTimeout(r, 2000));
    job = await client.jobs.get(job.id);
}
````

```go Go
ctx := context.Background()
client, _ := retab.NewClient("")

job, err := client.Jobs.Create(ctx, &retab.JobsCreateParams{
	Endpoint: retab.CreateJobRequestEndpointV1Extractions,
	Request:  retab.Resource{},
})
if err != nil {
	log.Fatal(err)
}
jobID := *job.ID

start := time.Now()
timeout := 60 * time.Second
terminal := map[string]bool{"completed": true, "failed": true, "cancelled": true, "expired": true}

for {
	status := string(*job.Status)
	if terminal[status] {
		break
	}
	if time.Since(start) > timeout {
		if _, err := client.Jobs.Cancel(ctx, jobID); err != nil {
			log.Fatal(err)
		}
		fmt.Println("Job cancelled due to timeout")
		break
	}
	time.Sleep(2 * time.Second)
	job, err = client.Jobs.Get(ctx, jobID, nil)
	if err != nil {
		log.Fatal(err)
	}
}
```

```rust Rust
use retab::enums::{CreateJobRequestEndpoint, JobStatus};
use retab::models::CreateJobRequest;
use retab::resources::jobs::{CreateParams, GetParams};
use retab::Retab;
use std::collections::HashMap;
use std::time::{Duration, Instant};

let client = Retab::new(std::env::var("RETAB_API_KEY")?);

let mut job = client.jobs().create(CreateParams::new(CreateJobRequest {
    endpoint: CreateJobRequestEndpoint::V1Extractions,
    request: HashMap::new(),
    metadata: None,
})).await?;

let job_id = job.id.clone();
let start = Instant::now();
let timeout = Duration::from_secs(60);

while !matches!(
    job.status,
    Some(JobStatus::Completed | JobStatus::Failed | JobStatus::Cancelled | JobStatus::Expired)
) {
    if start.elapsed() > timeout {
        client.jobs().cancel(&job_id).await?;
        println!("Job cancelled due to timeout");
        break;
    }
    tokio::time::sleep(Duration::from_secs(2)).await;
    job = client.jobs().get(&job_id, GetParams::default()).await?;
}
```

```php PHP
        break;

}
```
