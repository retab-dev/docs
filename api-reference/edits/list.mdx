---
title: 'List Edits'
openapi: 'GET /v1/edits'
---

List persisted edits for your organization, with cursor-based pagination and optional template filtering.

<RequestExample>

```python python
from retab import Retab

client = Retab()

edits = client.edits.list(limit=20, order="desc")
for edit in edits.data:
    print(f"{edit.id}: {edit.file.filename}")

# Filter by template
edits = client.edits.list(template_id="edittplt_abc123", limit=50)
```

```javascript javascript
import { Retab } from '@retab/node';

const client = new Retab();

const edits = await client.edits.list({ limit: 20, order: "desc" });
for (const edit of edits.data) {
    console.log(`${edit.id}: ${edit.file.filename}`);
}
```

```bash curl
curl -X GET \
  'https://api.retab.com/v1/edits?limit=20&order=desc' \
  -H "Api-Key: $RETAB_API_KEY"

# With a template_id filter
curl -X GET \
  'https://api.retab.com/v1/edits?template_id=edittplt_abc123' \
  -H "Api-Key: $RETAB_API_KEY"
```

</RequestExample>

## Query Parameters

<ParamField query="before" type="string">
  Cursor for backward pagination.
</ParamField>

<ParamField query="after" type="string">
  Cursor for forward pagination.
</ParamField>

<ParamField query="limit" type="integer" default="10">
  Maximum number of edits to return. Max: `100`.
</ParamField>

<ParamField query="order" type="string" default="desc">
  Sort order by `created_at`. Valid values: `asc`, `desc`.
</ParamField>

<ParamField query="filename" type="string">
  Case-insensitive substring filter applied to `file.filename`.
</ParamField>

<ParamField query="template_id" type="string">
  Only return edits created from this template.
</ParamField>

<ParamField query="from_date" type="string">
  Return edits created on or after this ISO date.
</ParamField>

<ParamField query="to_date" type="string">
  Return edits created on or before this ISO date.
</ParamField>
