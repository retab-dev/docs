---
title: 'Create Edit'
openapi: 'POST /v1/edits'
---

Automatically detect fillable regions in a document (or reuse a pre-defined template) and apply the values described in `instructions`. Persisted as an `Edit` resource, retrievable via `GET /v1/edits/{edit_id}`.

Either `document` or `template_id` must be provided; they are mutually exclusive.

<RequestExample>

```python python
from retab import Retab

client = Retab()

edit = client.edits.create(
    instructions="Name: John Doe, Date of Birth: 1990-01-15, Address: 123 Main Street",
    document="form.pdf",
    model="retab-small",
)

print(f"Edit ID: {edit.id}")
for field in edit.data.form_data:
    print(f"{field.key} = {field.value}")

# Save the filled PDF
import base64
with open("filled_form.pdf", "wb") as f:
    f.write(base64.b64decode(edit.data.filled_document.url.split(",", 1)[1]))
```


## Request Body

<ParamField body="instructions" type="string" required>
  Natural-language instructions describing how to fill the form.
</ParamField>

<ParamField body="document" type="MIMEData">
  Input document (PDF, DOCX, XLSX, or PPTX). Mutually exclusive with `template_id`.
</ParamField>

<ParamField body="template_id" type="string">
  `EditTemplate` id to fill. When provided, uses the template's pre-defined form fields. Mutually exclusive with `document`.
</ParamField>

<ParamField body="model" type="string" default="retab-small">
  The model used for edit inference.
</ParamField>

<ParamField body="config" type="EditConfig">
  Rendering configuration. Currently supports `color` (hex code of the color used for the filled text, default `#000080`).
</ParamField>

<ParamField body="bust_cache" type="boolean" default="false">
  When `true`, bypass the cache and re-run the edit.
</ParamField>

## Response Fields

<ResponseField name="id" type="string">
  Unique edit identifier.
</ResponseField>

<ResponseField name="file" type="object">
  File metadata for the source document or template PDF.
</ResponseField>

<ResponseField name="template_id" type="string | null">
  Template id when the edit was created from a template; `null` for direct-document edits.
</ResponseField>

<ResponseField name="data" type="object">
  The edit result.
  <Expandable title="properties">
    <ResponseField name="form_data" type="array[FormField]">
      Filled form fields (positions, descriptions, and values).
    </ResponseField>
    <ResponseField name="filled_document" type="MIMEData">
      PDF with the filled values rendered in.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="usage" type="RetabUsage">
  Page count and credit consumption.
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 creation timestamp.
</ResponseField>

<ResponseField name="updated_at" type="string">
  ISO 8601 last update timestamp.
</ResponseField>
