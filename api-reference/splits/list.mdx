---
title: 'List Splits'
openapi: 'GET /v1/splits'
---

List persisted splits for your organization, with cursor-based pagination.

<RequestExample>

```python python
from retab import Retab

client = Retab()

splits = client.splits.list(limit=20, order="desc")
for s in splits.data:
    print(f"{s.id}: {s.file.filename}")
```

```javascript javascript
import { Retab } from '@retab/node';

const client = new Retab();

const splits = await client.splits.list({ limit: 20, order: "desc" });
for (const s of splits.data) {
    console.log(`${s.id}: ${s.file.filename}`);
}
```

```bash curl
curl -X GET \
  'https://api.retab.com/v1/splits?limit=20&order=desc' \
  -H "Api-Key: $RETAB_API_KEY"
```

</RequestExample>

## Query Parameters

<ParamField query="before" type="string">
  Cursor for backward pagination.
</ParamField>

<ParamField query="after" type="string">
  Cursor for forward pagination.
</ParamField>

<ParamField query="limit" type="integer" default="10">
  Maximum number of splits to return.
</ParamField>

<ParamField query="order" type="string" default="desc">
  Sort order by `created_at`. Valid values: `asc`, `desc`.
</ParamField>

<ParamField query="filename" type="string">
  Case-insensitive substring filter applied to `file.filename`.
</ParamField>

<ParamField query="from_date" type="string">
  Return splits created on or after this date, in `YYYY-MM-DD` format.
</ParamField>

<ParamField query="to_date" type="string">
  Return splits created on or before this date, in `YYYY-MM-DD` format.
</ParamField>
