---
title: 'Create Split'
openapi: 'POST /v1/splits'
---

Split a multi-page document into labeled subdocuments and return the canonical split resource. This endpoint is split-only: key-based grouping belongs to [`/v1/partitions`](/api-reference/partitions/create).

<RequestExample>

```python python
from retab import Retab

client = Retab()

split = client.splits.create(
    document="invoice_batch.pdf",
    model="retab-small",
    subdocuments=[
        {
            "name": "invoice",
            "description": "Invoice documents with billing information",
            "allow_multiple_instances": True,
        },
        {"name": "receipt", "description": "Receipt documents for payments"},
        {"name": "contract", "description": "Legal contract documents"},
    ],
    instructions="Processing Q4 2024 vendor document batch",
    n_consensus=3,
    bust_cache=False,
)

for sub in split.output:
    print(f"{sub.name}: pages {sub.pages}")
```


## Request Body

<ParamField body="document" type="MIMEData" required>
  The document to split.
</ParamField>

<ParamField body="subdocuments" type="array[Subdocument]" required>
  The labeled subdocuments to split the document into. Each entry has `name`, optional `description`, and `allow_multiple_instances`.
</ParamField>

<ParamField body="model" type="string" default="retab-small">
  The model used for the split operation.
</ParamField>

<ParamField body="instructions" type="string">
  Free-form instructions appended to the system prompt to steer the split.
</ParamField>

<ParamField body="n_consensus" type="integer" default="1">
  Number of consensus split runs to perform. Max: `8`.
</ParamField>

<ParamField body="bust_cache" type="boolean" default="false">
  When `true`, bypass the cache and force a fresh split run.
</ParamField>

## Response Fields

<ResponseField name="id" type="string">
  Unique split identifier.
</ResponseField>

<ResponseField name="file" type="object">
  File metadata: `id`, `filename`, `mime_type`.
</ResponseField>

<ResponseField name="model" type="string">
  Model used for the split operation.
</ResponseField>

<ResponseField name="subdocuments" type="array[Subdocument]">
  The split labels used to produce this result.
</ResponseField>

<ResponseField name="n_consensus" type="integer">
  Number of consensus votes used to produce the result.
</ResponseField>

<ResponseField name="instructions" type="string | null">
  Free-form instructions supplied with the request.
</ResponseField>

<ResponseField name="output" type="array[SplitResult]">
  The list of subdocuments with their assigned pages.
  <Expandable title="properties">
    <ResponseField name="name" type="string">
      The subdocument label.
    </ResponseField>
    <ResponseField name="pages" type="array[integer]">
      Pages (1-indexed) assigned to this subdocument.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="consensus" type="SplitConsensus | null">
  Consensus metadata for multi-vote runs.
  <Expandable title="properties">
    <ResponseField name="likelihoods" type="array[SplitSubdocumentLikelihood] | null">
      Likelihoods aligned positionally with `output`.
    </ResponseField>
    <ResponseField name="choices" type="array[array[SplitResult]]">
      Alternative split vote outputs used to build the consolidated result.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="usage" type="RetabUsage | null">
  Usage information for the split operation.
</ResponseField>

<ResponseField name="created_at" type="string | null">
  ISO 8601 creation timestamp.
</ResponseField>

<ResponseField name="updated_at" type="string | null">
  ISO 8601 last update timestamp.
</ResponseField>
