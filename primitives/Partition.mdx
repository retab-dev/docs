### Introduction

The `partitions.create` method groups a document into repeated chunks based on a key such as `invoice_number`, `policy_id`, or `claim_number`. It is useful when one document contains multiple logical records of the same kind and you need one chunk per detected key value.

Unlike `split`, partitioning does not classify the document into different subdocument types. Unlike `parse`, `extract`, `split`, or `classify`, it also does not create a stored resource that you later retrieve or list. It returns partitioned chunks directly in the response.

Common use cases include:

1. **Invoice Batches**: Separate one PDF into one chunk per invoice number
2. **Claim Packets**: Group pages by claim ID inside a large insurance packet
3. **Policy Documents**: Break a carrier export into one chunk per policy number
4. **Repeated Records**: Segment documents containing many repeated forms of the same type

Key features of the Partition API:

- **Key-Based Grouping**: Group repeated records by a business key rather than by document type
- **Chunk Output**: Returns one extracted document chunk per detected key value
- **Page-Level Mapping**: Each chunk includes the 1-indexed `pages` it covers
- **Sub-Page Bounds**: Optional `y_start` and `y_end` identify partial-page boundaries when needed
- **No Persistence Layer**: The endpoint returns derived chunks directly instead of creating a retrievable resource

## Use Case: Partitioning An Invoice Batch

Use partitioning when every record is the same general document type, but each record has its own identifier and should become its own chunk.

<CodeGroup>
```python Python
from retab import Retab

client = Retab()

response = client.partitions.create(
    document="invoice_batch.pdf",
    key="invoice_number",
    instructions="Return one chunk per invoice number and keep all pages for the same invoice together.",
    model="retab-small",
)

for chunk in response.chunks:
    print(chunk.key_value, chunk.pages)
```

```javascript JavaScript
import { Retab } from '@retab/node';

const client = new Retab();

const response = await client.partitions.create({
  document: 'invoice_batch.pdf',
  key: 'invoice_number',
  instructions: 'Return one chunk per invoice number and keep all pages for the same invoice together.',
  model: 'retab-small',
});

for (const chunk of response.chunks) {
  console.log(chunk.key_value, chunk.pages);
}
```
</CodeGroup>

## When To Use Partition vs Split

- Use `partitions.create` when every record is conceptually the same kind of document and you want one chunk per repeated key value.
- Use `splits.create` when you need to classify a document into different subdocument types such as `invoice`, `receipt`, and `contract`.
- Use `partitions.create` for key-based grouping whenever repeated records need to be separated by identifiers such as invoice number, claim ID, or policy ID.
